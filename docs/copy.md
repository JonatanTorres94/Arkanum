# Arkanum — Landing Copy Foundation

## 1. Propósito del documento

Este documento define el copy base para la landing pública de Arkanum.

Debe ser usado como fuente principal para implementar la versión `v0.2.0 — Brand Landing MVP`.

La landing debe vender claridad, diagnóstico y solución operativa. No debe vender tecnología como mensaje principal.

### Título

```text
Cuando la operación crece, las herramientas improvisadas empiezan a fallar.
```

### Descripción


```text

```

```text
Información dispersa, mensajes perdidos y poca trazabilidad sobre lo que realmente pasó.
```

#### Stock sin control

```text
Diferencias entre lo vendido, lo disponible y lo registrado en el sistema.
```

```text

```

```text

```

#### Empleados sin app operativa


## 7. Método de trabajo

### Título

```text
Primero entendemos el proceso. Después construimos la solución.
```

### Descripción

```text
El software a medida solo tiene sentido cuando responde a un problema real. Por eso empezamos con diagnóstico, definimos alcance y avanzamos por etapas.
```

### Paso 1 — Diagnóstico

```text
Relevamos cómo funciona hoy el proceso, qué herramientas usan y dónde se pierden tiempo, datos o control.
```

### Paso 2 — Diseño de solución

```text
Definimos qué conviene automatizar, qué debe integrarse y cuál es el alcance mínimo para generar valor.
```

### Paso 3 — MVP funcional

```text
Construimos una primera versión usable, enfocada en resolver el problema principal sin sobredimensionar el proyecto.
```

### Paso 4 — Implementación

```text
Acompañamos la puesta en marcha, ajustes iniciales y adopción por parte del equipo.
```

### Paso 5 — Mejora continua

```text
Evolucionamos la solución según uso real, nuevas necesidades y crecimiento de la empresa.
```

## 13. Mensajes que deben evitarse

No usar frases genéricas como:

```text
Transformamos digitalmente tu negocio.
Creamos soluciones innovadoras.
Impulsamos el futuro con tecnología.
Somos tu partner digital.
```

Son vagas y no explican el valor.

Usar frases concretas:

```text
Reducimos carga manual.
Conectamos sistemas desconectados.
Ordenamos stock, pedidos y reportes.
Creamos apps para empleados en campo.
Automatizamos procesos repetitivos.
```

---

## 14. Regla final de copy

Cada sección debe responder una pregunta del cliente:

```text
¿Qué hacen?
¿Qué problema mío resuelven?
¿Qué tipo de soluciones construyen?
Cómo trabajan?
Por qué debería confiar?
Cómo doy el primer paso?
```

Si una sección no responde ninguna de esas preguntas, probablemente sobra.
