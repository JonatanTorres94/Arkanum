# Arkanum — Brand Foundation

## 1. Propósito del documento

Este documento define la base de marca para Arkanum.

Debe ser usado como referencia obligatoria antes de diseñar, escribir copy o implementar componentes visuales de la web pública.

Arkanum no debe nacer como un portfolio personal ni como una landing genérica de agencia. Debe posicionarse como una empresa técnica orientada a resolver problemas reales de operación mediante software a medida, automatización e integración de sistemas.

---

## 2. Nombre provisional

```text
Arkanum
```

El nombre se usará como identidad provisional para diseño, copy e implementación.

Todavía puede cambiar antes del lanzamiento público, por lo tanto:

- no hardcodear assets de marca de forma difícil de reemplazar;
- centralizar nombre y metadata en configuración;
- evitar dependencias visuales imposibles de adaptar si cambia el nombre.

Archivo sugerido:

```text
src/config/site.ts
```

---

## 3. Posicionamiento

### Versión principal

```text
Automatizamos y modernizamos procesos de tu empresa con sistemas web, apps móviles e integraciones a medida.
```

### Versión extendida

```text
Creamos soluciones digitales para reemplazar planillas desordenadas, tareas repetitivas, sistemas desconectados y procesos lentos por herramientas simples, escalables y adaptadas a la operación real de cada empresa.
```

### Concepto central

```text
Arkanum transforma procesos operativos deficientes en sistemas digitales claros, medibles y preparados para crecer.
```

---

## 4. Qué problema resuelve Arkanum

Arkanum apunta a empresas que operan con fricción interna.

Dolores principales:

- Excel descontrolado.
- Carga manual duplicada.
- Stock sin control.
- Pedidos por WhatsApp.
- Conciliación de pagos lenta o manual.
- Reportes hechos a mano.
- Falta de trazabilidad.
- Apps internas lentas, inexistentes o mal diseñadas.
- Sistemas legacy que no se conectan entre sí.
- Empleados en campo sin herramientas móviles para registrar actividad.

La categoría mental no es “desarrollo de software”.

La categoría mental es:

```text
Automatización operativa y modernización de procesos.
```

---

## 5. Cliente objetivo inicial

Arkanum no se limita a un nicho único, pero la comunicación inicial debe priorizar empresas con procesos operativos claros.

### Clientes prioritarios

1. Distribuidoras.
2. Logística y repartos.
3. Comercios con stock / mayoristas.
4. E-commerce.
5. Profesionales y consultoras con procesos administrativos repetitivos.

### Segmento principal para la web

```text
Empresas con operación, stock, pedidos, administración o personal en movimiento que necesitan ordenar procesos y reducir carga manual.
```

### Segmento secundario

```text
Negocios digitales o administrativos que necesitan centralizar información, automatizar tareas y obtener reportes confiables.
```

---

## 6. Servicios iniciales

La web debe comunicar tres líneas principales.

### 6.1. Sistemas web administrativos

Paneles internos para gestionar pedidos, clientes, stock, tareas, operaciones, reportes y procesos administrativos.

Ejemplos:

- panel de pedidos;
- gestión de stock;
- administración de clientes;
- control de operaciones;
- reportes internos;
- herramientas para equipos administrativos.

---

### 6.2. Apps móviles operativas

Aplicaciones para empleados, técnicos, repartidores, preventistas o personal en campo.

Casos típicos:

- registrar visitas;
- reportar tareas;
- cargar stock;
- confirmar entregas;
- registrar ubicación o actividad;
- trabajar con conectividad limitada;
- sincronizar información con sistemas internos.

---

### 6.3. Dashboards de gestión

Tableros para visualizar métricas relevantes y tomar decisiones con datos.

Casos típicos:

- ventas;
- stock;
- pagos;
- tareas;
- productividad;
- operaciones;
- reportes gerenciales;
- indicadores por período.

---

## 7. Integraciones

Las integraciones no serán presentadas como servicio aislado principal, sino como capacidad transversal.

Arkanum puede conectar:

- sistemas internos;
- APIs externas;
- e-commerce;
- bancos;
- Mercado Pago;
- planillas;
- CRMs;
- ERPs;
- herramientas legacy.

Mensaje recomendado:

```text
Conectamos tus herramientas actuales para evitar carga duplicada y centralizar información.
```

---

## 8. Oferta de entrada

La oferta inicial será:

```text
Diagnóstico inicial sin cargo
```

Debe presentarse como una instancia consultiva, no como una llamada de venta genérica.

Mensaje:

```text
Analizamos tu proceso actual, detectamos oportunidades de automatización y te indicamos si una solución a medida tiene sentido para tu empresa.
```

Regla comercial:

```text
No todo problema merece software custom. Arkanum debe transmitir criterio, no desesperación por vender.
```

---

## 9. Tono de comunicación

Arkanum debe sonar:

- claro;
- serio;
- técnico pero entendible;
- consultivo;
- directo;
- orientado a negocio;
- confiable;
- moderno;
- sin humo.

Debe evitar sonar:

- excesivamente corporativo;
- críptico;
- esotérico;
- gamer;
- futurista vacío;
- freelance improvisado;
- agencia de marketing;
- template SaaS genérico.

- reportes;

## 11. Uso de IA en la marca

Arkanum puede mencionar uso de herramientas modernas e IA, pero no debe vender “magia con IA”.

Mensaje permitido:

```text
Combinamos criterio técnico humano con herramientas modernas de desarrollo asistido para acelerar entregas sin sacrificar calidad.
```

Evitar:

```text
Creamos sistemas con inteligencia artificial en tiempo récord.
```

El diferencial no es “usar IA”.

El diferencial es:

```text
Entender el proceso, diseñar bien la solución y construir software mantenible con mayor velocidad operativa.
```

---

## 12. Personalidad visual

Estilo aprobado:

```text
Minimalista premium, oscuro/elegante, con lenguaje simple para pymes.
```

La web debe transmitir:

- precisión;
- profundidad técnica;
- confianza;
- orden;
- claridad;
- modernidad;
- capacidad de ejecución.

No debe parecer:

- cyberpunk excesivo;
- sitio gamer;
- landing cripto;
- portfolio personal;
- agencia creativa genérica;
- consultora corporativa fría;
- producto místico.

---

## 13. Paleta visual tentativa

Paleta sugerida:

```text
Background principal: negro azulado profundo / grafito
Superficies: gris carbón
Texto principal: blanco cálido o gris muy claro
Texto secundario: gris medio
Acento principal: cian tecnológico o azul eléctrico controlado
Acento secundario opcional: dorado sutil
Bordes: gris oscuro translúcido
```

Regla:

```text
El acento debe guiar la conversión, no decorar sin sentido.
```

No abusar de gradientes ni brillos.

---

## 14. Tipografía

La tipografía debe ser moderna, legible y sobria.

Criterios:

- buena lectura en mobile;
- títulos fuertes;
- párrafos claros;
- no usar fuentes excesivamente futuristas;
- no sacrificar legibilidad por estética.

---

## 15. Diseño de componentes

Los componentes deben verse:

- limpios;
- consistentes;
- con buen espaciado;
- con bordes sutiles;
- con jerarquía clara;
- sin saturación visual.

Patrones recomendados:

- cards oscuras con borde sutil;
- badges para categorías;
- CTAs visibles;
- secciones amplias;
- iconografía lineal simple;
- máximo 1 o 2 efectos visuales distintivos.

---

## 16. CTA principal

CTA aprobado:

```text
Diagnosticar mi proceso sin cargo
```

Alternativas válidas:

```text
Solicitar diagnóstico
Quiero analizar mi proceso
Contar mi problema
```

Evitar:

```text
Contacto
Enviar
Click aquí
Más información
```

---

## 17. Promesa comercial

Promesa permitida:

```text
Reducir carga manual, ordenar información y mejorar el control operativo mediante software a medida.
```

No prometer:

- resultados garantizados;
- ahorros exactos sin diagnóstico;
- tiempos imposibles;
- automatización total sin análisis;
- reemplazo completo de personal;
- éxito asegurado.

---

## 18. Confianza técnica

Arkanum puede mencionar:

- arquitectura limpia;
- seguridad;
- escalabilidad;
- documentación;
- control de accesos;
- bases de datos robustas;
- despliegues preparados para crecer;
- mantenimiento evolutivo.

Pero debe hacerlo en lenguaje comprensible.

Ejemplo:

```text
Construimos soluciones pensadas para crecer: con arquitectura clara, control de accesos, documentación y bases de datos robustas.
```

---

## 19. Qué no debe decir Arkanum

Evitar claims como:

```text
Somos líderes del mercado.
Tenemos años de experiencia comprobada.
Trabajamos con las mejores empresas.
Garantizamos resultados.
Somos expertos certificados.
```

Si no hay evidencia real, no se usa.

La confianza debe construirse desde:

- claridad;
- criterio;
- proceso;
- profesionalismo;
- honestidad;
- calidad técnica.

---

## 20. Regla final de marca

Arkanum debe comunicar una idea simple:

```text
Si tu empresa está creciendo pero tus procesos siguen dependiendo de planillas, WhatsApp o sistemas viejos, podemos ayudarte a ordenar, automatizar y modernizar la operación con software a medida.
```
